# TPI Metodología de Sistemas II - Body Paint

Este repositorio contiene el Trabajo Práctico Integrador (TPI) de la materia Metodología de Sistemas II.

## 🎯 Objetivo General
Diseñar y desarrollar un producto de software web para la empresa **Body Paint**, aplicando conceptos de metodologías ágiles (Scrum, User Stories, MVP, desarrollo iterativo e incremental).

## 🏢 Dominio del Problema
**Body Paint** vende artículos para body painting. Necesitan una plataforma web para gestionar la venta online de sus productos.

**Roles principales del sistema:**
1. **Cliente:** Se registra, agrega productos a un carro de compras, aplica cupones de descuento y confirma pedidos (con distintas formas de pago).
2. **Vendedor:** Visualiza el estado de los pedidos, gestiona entregas y genera cupones de descuento. También accede a reportes de ventas.
3. **Administrador de Productos:** Gestiona el stock, carga masivamente productos (csv/xls), crea "Kits" (conjunto de productos) y visualiza alertas de stock mínimo.

## 🚀 Alcance del MVP (Mínimo Producto Viable)
El sistema se construirá de manera iterativa. Para el primer nivel de completitud (MVP obligatorio definido por la cátedra), se deben implementar como mínimo las siguientes funcionalidades:

- **Rol Cliente:**
  - Registrar cliente
  - Armar carro de compras
  - Confirmar pedido con pago en efectivo contra entrega

- **Rol Vendedor:**
  - Visualizar estado de pedidos

- **Rol Administrador de productos:**
  - Visualizar productos
  - Crear producto

## 🛠 Metodología de Trabajo
- **Framework:** Scrum con Sprints de 2 semanas.
- **Herramientas de Gestión:** Jira Software para gestión de tareas, Backlog y Sprints.
- **Control de Versiones:** GitHub.

## 📅 Evaluación y Entrega
El proyecto será evaluado en base a la calidad del análisis (User Stories), la correcta aplicación de Scrum, el correcto funcionamiento de los entregables en cada Sprint, y la claridad en las presentaciones (Sprint Reviews).
